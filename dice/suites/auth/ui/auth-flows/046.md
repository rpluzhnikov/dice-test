## Steps

1. До входа проверить производное состояние аутентификации
   - Expected: isAuthenticated равно false
   - Expected: currentUsername пустой
2. Войти с корректными данными (или зарегистрироваться)
   - Expected: isAuthenticated равно true
   - Expected: currentUsername соответствует имени вошедшего пользователя
3. Выйти из системы
   - Expected: isAuthenticated равно false
   - Expected: currentUsername снова пустой

## Notes

Цикл производного состояния §6.1: round-trip isAuthenticated (false → true → false) и currentUsername (пусто → имя → пусто). Состояние наблюдается через интерфейс (доступность авторизованных элементов, отображаемое имя).
