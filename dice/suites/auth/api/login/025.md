## Steps

1. Отправить серию (не менее 30) запросов POST /api/v1/login с телом {"username":"ghost_user_0001","password":"anyPassw0rd"} и зафиксировать время ответа каждого
   - Expected: Все ответы 401 с code INVALID_CREDENTIALS
   - Expected: Получено распределение времён ответа для сценария «нет такого пользователя»
2. Отправить серию (не менее 30) запросов POST /api/v1/login с телом {"username":"user1","password":"wrongPassw0rd"} и зафиксировать время ответа каждого
   - Expected: Все ответы 401 с code INVALID_CREDENTIALS
   - Expected: Получено распределение времён ответа для сценария «неверный пароль»
3. Сравнить медианы и распределения времён ответа двух серий
   - Expected: Между сериями нет существенной (стабильно воспроизводимой) разницы во времени ответа

## Notes

needs-tooling: требуется измерение времени на повторяющихся выборках (одиночный замер недостоверен). Если на пути «нет пользователя» отсутствует сравнение с фиктивным хешем (dummy-hash compare), вероятна timing-разница — отметить как находку для продукта.
