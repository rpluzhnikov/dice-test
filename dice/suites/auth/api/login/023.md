## Steps

1. Отправить POST /api/v1/login с телом {"username":"ghost_user_0001","password":"anyPassw0rd"} (несуществующий пользователь)
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code INVALID_CREDENTIALS
2. Отправить POST /api/v1/login с телом {"username":"user1","password":"wrongPassw0rd"} (существующий пользователь, неверный password)
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code INVALID_CREDENTIALS
3. Сравнить ответы шагов 1 и 2 между собой
   - Expected: HTTP-статус совпадает (401 в обоих)
   - Expected: Значение code совпадает (INVALID_CREDENTIALS в обоих)
   - Expected: Структура тела ответа совпадает
   - Expected: Ни одно поле ответа не указывает, существует ли пользователь

## Notes

Ядро проверки anti-enumeration: ответ при неизвестном пользователе и при неверном пароле байт-идентичен (одинаковый статус, code и форма тела). Decision table по сценариям входа.
