## Steps

1. Отправить POST /api/v1/login с телом {"username":"<существующий username>","password":"<корректный password>"}
   - Expected: Ответ 200 OK
   - Expected: Тело ответа содержит поле token
   - Expected: Значение token непустое
   - Expected: token состоит из трёх сегментов, разделённых точками (формат header.payload.signature)
2. Сравнить набор полей тела ответа со спецификацией
   - Expected: Тело ответа содержит единственное поле token
   - Expected: Тело не содержит userId, username, пароль/хеш пароля, email и иные данные пользователя

## Notes

Объединяет успешный вход (пункт 1 чек-листа) и проверку отсутствия утечки данных в теле ответа (пункт 17): при 200 наружу отдаётся только token.
