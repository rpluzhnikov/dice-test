## Steps

1. Отправить POST /api/v1/login с телом {"username":"","password":"abcdef12"} (пустой username)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_BODY
2. Отправить POST /api/v1/login с телом {"username":"user1","password":""} (пустой password)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_BODY
3. Отправить POST /api/v1/login с телом {"username":"","password":""} (оба поля пустые)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_BODY
4. Отправить POST /api/v1/login с телом {"password":"abcdef12"} (отсутствует ключ username)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_BODY
5. Отправить POST /api/v1/login с телом {"username":"user1"} (отсутствует ключ password)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_BODY

## Notes

Спецификация §4: пустые или отсутствующие поля тела отклоняются как INVALID_BODY. Консолидация вариантов EP «невалидное тело» в один кейс.
