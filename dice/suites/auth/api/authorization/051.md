## Steps

1. Сформировать корректно подписанный (серверным секретом) JWT с claim exp в прошлом (например, exp = текущее время минус 1 час) и валидными sub/iat
   - Expected: Получен синтаксически корректный токен с истёкшим сроком действия
2. Отправить GET /api/v1/wallet с заголовком Authorization: Bearer <токен из шага 1>
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED

## Notes

needs-tooling: токен с exp в прошлом формируется генератором. Реальный TTL 7 дней нельзя дождаться по wall-clock, поэтому истечение моделируется значением exp.
