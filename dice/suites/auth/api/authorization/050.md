## Steps

1. Сформировать корректно подписанный (серверным секретом) JWT с пустым claim sub ("sub":"") и отправить GET /api/v1/wallet с заголовком Authorization: Bearer <этот токен>
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED
2. Сформировать корректно подписанный JWT БЕЗ claim sub (поле отсутствует) и отправить GET /api/v1/wallet с заголовком Authorization: Bearer <этот токен>
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED

## Notes

needs-tooling: требуется JWT-подписывающий помощник с серверным JWT_SECRET. Подпись валидна, но без идентифицируемого пользователя (пустой/отсутствующий sub) middleware не может разрешить userId и обязан отклонить запрос.
