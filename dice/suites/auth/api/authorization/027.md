## Steps

1. Отправить POST /api/v1/login с корректными username и password и получить token
   - Expected: Ответ 200 OK с непустым token
2. Отправить GET /api/v1/wallet с заголовком Authorization: Bearer <token из шага 1>
   - Expected: Ответ 200 OK
   - Expected: Запрос обработан как от вошедшего пользователя (userId извлечён из токена)

## Notes

Round-trip: выданный при логине токен принимается middleware на защищённом маршруте. GET /api/v1/wallet используется как представитель защищённых маршрутов.
