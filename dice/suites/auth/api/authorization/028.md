## Steps

1. Отправить GET /api/v1/wallet без заголовка Authorization
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED
2. Отправить GET /api/v1/wallet с заголовком Authorization: Token <валидный token> (без префикса Bearer)
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED
3. Отправить GET /api/v1/wallet с заголовком Authorization: <валидный token> (сырой токен без схемы)
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED
4. Отправить GET /api/v1/wallet с заголовком Authorization: Bearer (префикс есть, токен после него пустой)
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED

## Notes

Middleware требует заголовок строго в формате "Bearer <token>". GET /api/v1/wallet — представитель защищённых маршрутов.
