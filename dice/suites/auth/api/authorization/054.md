## Steps

1. Отправить POST /api/v1/login с корректными username и password и дополнительным заголовком Authorization: Bearer <валидный token>
   - Expected: Ответ 200 OK
   - Expected: Тело ответа содержит поле token
   - Expected: Ответ не равен 401 и не равен 500

## Notes

Публичные маршруты не требуют токена; наличие валидного токена на login не должно ломать обработку — токен игнорируется без ошибки.
