## Steps

1. Сформировать неподписанный JWT с header alg none и корректными claims (sub/iat/exp) и отправить GET /api/v1/wallet с заголовком Authorization: Bearer <этот токен>
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED
2. Сформировать JWT с header alg RS256 (алгоритм, отличный от HS256) и корректными claims и отправить GET /api/v1/wallet с заголовком Authorization: Bearer <этот токен>
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED

## Notes

needs-tooling: требуется генератор токенов с alg none и не-HS256. Алгоритм проверки жёстко зафиксирован на HS256; подмена alg (alg-confusion) не должна обходить проверку подписи.
