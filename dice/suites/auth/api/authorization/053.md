## Steps

1. Отправить GET /api/v1/wallet без заголовка Authorization
   - Expected: Ответ 401 Unauthorized с code UNAUTHORIZED
2. Отправить GET /api/v1/wallet с заголовком Authorization: Bearer <валидный token>
   - Expected: Ответ не равен 401 (запрос доходит до обработчика)
3. Отправить POST /api/v1/wallet/daily без заголовка Authorization
   - Expected: Ответ 401 Unauthorized с code UNAUTHORIZED
4. Отправить POST /api/v1/wallet/daily с заголовком Authorization: Bearer <валидный token>
   - Expected: Ответ не равен 401 (запрос доходит до обработчика)
5. Отправить GET /api/v1/leaderboard без заголовка Authorization
   - Expected: Ответ 401 Unauthorized с code UNAUTHORIZED
6. Отправить GET /api/v1/leaderboard с заголовком Authorization: Bearer <валидный token>
   - Expected: Ответ не равен 401 (запрос доходит до обработчика)
7. Отправить запрос к представителю GET /api/v1/games/... без заголовка Authorization
   - Expected: Ответ 401 Unauthorized с code UNAUTHORIZED
8. Отправить тот же запрос GET /api/v1/games/... с заголовком Authorization: Bearer <валидный token>
   - Expected: Ответ не равен 401 (запрос доходит до обработчика)
9. Отправить запрос к представителю POST /api/v1/games/... без заголовка Authorization
   - Expected: Ответ 401 Unauthorized с code UNAUTHORIZED
10. Отправить тот же запрос POST /api/v1/games/... с заголовком Authorization: Bearer <валидный token>
   - Expected: Ответ не равен 401 (запрос доходит до обработчика)

## Notes

Матрица доступа по представителю каждой защищённой группы (wallet, wallet/daily, leaderboard, games GET/POST): без токена — 401 UNAUTHORIZED, с валидным токеном — запрос доходит до обработчика (не 401).
ASSUMPTION: точные подпути /api/v1/games/... не зафиксированы в брифе — берётся по одному представителю на метод; конкретные пути уточнить по спецификации маршрутов.
