## Steps

1. Сформировать JWT с валидной структурой (alg HS256, корректные claims sub/iat/exp), подписав его НЕВЕРНЫМ секретом (отличным от серверного JWT_SECRET)
   - Expected: Получен синтаксически корректный токен из трёх сегментов
2. Отправить GET /api/v1/wallet с заголовком Authorization: Bearer <токен из шага 1>
   - Expected: Ответ 401 Unauthorized
   - Expected: Тело ответа содержит code UNAUTHORIZED

## Notes

needs-tooling: требуется JWT-подписывающий помощник с секретом, отличным от серверного JWT_SECRET. Структура токена валидна, подпись — нет; middleware обязан отклонить по несовпадению подписи.
