## Steps

1. Отправить POST /api/v1/register с телом {"password":"abcdef12"} (поле username отсутствует)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
2. Отправить POST /api/v1/register с телом {"username":"abcdef"} (поле password отсутствует)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code WEAK_PASSWORD
3. Отправить POST /api/v1/register с телом {"username":null,"password":"abcdef12"} (username = null)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
4. Отправить POST /api/v1/register с телом {"username":"abcdef","password":null} (password = null)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code WEAK_PASSWORD
5. Отправить POST /api/v1/register с телом {} (пустой объект)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME

## Notes

Тело является валидным JSON-объектом, поэтому INVALID_BODY не применяется; валидация полей идёт в порядке §3.2: сначала username (INVALID_USERNAME), затем password (WEAK_PASSWORD). Для пустого объекта первым проваливается username.

ASSUMPTION: отсутствующее/null поле трактуется так же, как невалидное значение того же поля (INVALID_USERNAME / WEAK_PASSWORD), а не как INVALID_BODY. Уточнить с продуктом: должно ли отсутствие обязательного поля давать INVALID_BODY вместо кода конкретного поля.
