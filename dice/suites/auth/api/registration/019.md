## Steps

1. Отправить POST /api/v1/register с телом {"username":"massassign","password":"abcdef12","role":"admin","chips":99999,"userId":"u_x","isAdmin":true} (валидные username/password + привилегированные поля)
   - Expected: Ответ 201 Created
   - Expected: Тело ответа содержит поле userId, сгенерированное сервером (не равное "u_x")
2. Залогиниться как massassign через POST /api/v1/login и запросить GET /wallet с полученным токеном
   - Expected: Баланс равен 1000 фишек (значение по умолчанию, а не 99999)
3. Выполнить от имени massassign действие, требующее обычных прав, и действие, требующее прав администратора
   - Expected: Привилегированные поля role/isAdmin из тела запроса проигнорированы; пользователь не получил роль admin

## Notes

Защита от mass-assignment: сервер принимает из тела только username и password, остальные поля (role, chips, userId, isAdmin) игнорирует. Аккаунт создаётся со значениями по умолчанию — стартовый баланс 1000 фишек, серверный userId, обычная роль.
