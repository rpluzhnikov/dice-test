## Steps

1. Отправить POST /api/v1/register с телом {"username":"statsuser","password":"abcdef12"} (валидный запрос)
   - Expected: Ответ 201 Created
   - Expected: Тело ответа содержит поле userId
2. Запросить GET /leaderboard
   - Expected: Ответ 200 OK
   - Expected: Новый пользоватеь присутствует в выдаче с показателями wins=0, games_played=0, net_chips=0

## Notes

Проверяет побочный эффект: при регистрации создаётся запись user_stats с нулевыми показателями (wins=0, games_played=0, net_chips=0).

ASSUMPTION: leaderboard включает пользователей с нулевыми результатами. Если новый пользователь без сыгранных игр не отображается в leaderboard, показатели нельзя проверить по чёрному ящику — тогда кейс выполняется как документированный контракт и требует доступа к БД/админ-чтению (тег needs-tooling).
