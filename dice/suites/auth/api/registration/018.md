## Steps

1. Отправить POST /api/v1/register с телом {"username":"CaseUser","password":"abcdef12"}
   - Expected: Ответ 201 Created
   - Expected: Тело ответа содержит поле userId
2. Отправить POST /api/v1/register с телом {"username":"caseuser","password":"abcdef12"} (тот же набор букв, нижний регистр)
   - Expected: Ответ 201 Created
   - Expected: Тело ответа содержит поле userId с другим значением, чем на шаге 1

## Notes

Ограничение уникальности username чувствительно к регистру (§7). CaseUser и caseuser считаются разными именами, обе регистрации успешны и создают разные аккаунты.
