## Steps

1. Отправить POST /api/v1/register с телом {"username":"a' OR '1'='1","password":"abcdef12"} (SQL-инъекция в username)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
   - Expected: Никакой пользователь не создан, признаков выполнения инъекции нет
2. Отправить POST /api/v1/register с телом {"username":"admin\"--","password":"abcdef12"} (попытка комментирования SQL в username)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
   - Expected: Никакой пользователь не создан, признаков выполнения инъекции нет

## Notes

Инъекционные строки содержат недопустимые для username символы (кавычки, пробелы, дефис), поэтому отклоняются валидацией как INVALID_USERNAME ещё до обращения к хранилищу. Побочного эффекта инъекции быть не должно.
