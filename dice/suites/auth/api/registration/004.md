## Steps

1. Отправить POST /api/v1/register с телом {"username":"ab-cd","password":"abcdef12"} (дефис)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
2. Отправить POST /api/v1/register с телом {"username":"ab.cd","password":"abcdef12"} (точка)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
3. Отправить POST /api/v1/register с телом {"username":"ab cd","password":"abcdef12"} (пробел внутри)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
4. Отправить POST /api/v1/register с телом {"username":" abcd","password":"abcdef12"} (ведущий пробел)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
   - Expected: Пробел не отсекается автоматически, регистрация не выполняется
5. Отправить POST /api/v1/register с телом {"username":"abcd ","password":"abcdef12"} (завершающий пробел)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
   - Expected: Пробел не отсекается автоматически, регистрация не выполняется
6. Отправить POST /api/v1/register с телом {"username":"abcdé","password":"abcdef12"} (unicode-буква é)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
7. Отправить POST /api/v1/register с телом {"username":"abcd😀","password":"abcdef12"} (emoji)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME

## Notes

Error guessing по таксономии недопустимых символов username (разрешены только A–Z/a–z/0–9/_). Шаги с ведущим/завершающим пробелом дополнительно подтверждают отсутствие неявного trim.
