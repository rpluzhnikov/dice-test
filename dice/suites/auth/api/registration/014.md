## Steps

1. Отправить POST /api/v1/register с телом {"username":"newuser01","password":"abcdef12"} (валидный запрос)
   - Expected: Ответ 201 Created
   - Expected: Тело ответа содержит поле userId со значением, начинающимся с префикса u_
2. Изучить тело ответа 201 целиком
   - Expected: В теле отсутствуют поля token, jwt, access_token, refresh_token, session
   - Expected: В теле отсутствует значение password (открытым текстом или в любом виде)
3. Отправить POST /api/v1/register с телом {"username":"ab","password":"abcdef12"} (невалидный username, ответ 400)
   - Expected: Ответ 400 Bad Request
   - Expected: В теле ответа отсутствует поле password и его значение
4. Зарегистрировать username из шага 1 повторно тем же телом (ответ 409)
   - Expected: Ответ 409 Conflict
   - Expected: В теле ответа отсутствует поле password и его значение

## Notes

Объединяет проверку happy-path успеха с двумя проверками безопасности: (а) в ответе регистрации не возвращается токен/сессия — токен выдаётся только при логине; (б) password никогда не присутствует в теле ответа ни при 201, ни при 400, ни при 409 (поглощённый пункт 21 чек-листа).
