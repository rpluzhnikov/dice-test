## Steps

1. Отправить POST /api/v1/register с телом {"username":"duptest_user","password":"abcdef12"} (первая регистрация)
   - Expected: Ответ 201 Created
   - Expected: Тело ответа содержит поле userId
2. Отправить POST /api/v1/register повторно с тем же телом {"username":"duptest_user","password":"abcdef12"}
   - Expected: Ответ 409 Conflict
   - Expected: Тело ответа содержит code USERNAME_TAKEN
3. Залогиниться как duptest_user и сверить, что аккаунт остался единственным
   - Expected: userId совпадает с полученным на шаге 1
   - Expected: Отклонённая попытка не создала второго пользователя и второй записи user_stats

## Notes

Имя пользователя уникально. Повторная регистрация занятого username отклоняется с 409 USERNAME_TAKEN и не приводит к созданию второго аккаунта или второй записи статистики.
