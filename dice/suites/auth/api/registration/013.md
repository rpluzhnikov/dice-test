## Steps

1. Отправить POST /api/v1/register с телом {"username":12345,"password":"abcdef12"} (username — число)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
2. Отправить POST /api/v1/register с телом {"username":["abc"],"password":"abcdef12"} (username — массив)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
3. Отправить POST /api/v1/register с телом {"username":{"v":"abc"},"password":"abcdef12"} (username — объект)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code INVALID_USERNAME
4. Отправить POST /api/v1/register с телом {"username":"abcdef","password":12345678} (password — число)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code WEAK_PASSWORD
5. Отправить POST /api/v1/register с телом {"username":"abcdef","password":true} (password — boolean)
   - Expected: Ответ 400 Bad Request
   - Expected: Тело ответа содержит code WEAK_PASSWORD

## Notes

Поля присутствуют, но имеют неверный тип. Тело — валидный JSON-объект, поэтому INVALID_BODY не применяется; нестроковое значение поля трактуется как невалидное значение этого поля (username → INVALID_USERNAME, password → WEAK_PASSWORD) по порядку §3.2.
