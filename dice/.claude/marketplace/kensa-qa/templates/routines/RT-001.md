id: RT-001
name: Smoke tour of the homepage
engine: claude
description: Opens the homepage, verifies the hero headline, and checks the primary nav links are present and clickable.
---

Smoke-check the site homepage. Use the `kensa-browser` skill's verb set.

1. Confirm the browser is reachable: `kensa-cli browser status --format json`.
   If it reports `reachable: false` or you get exit code 2, stop and tell the user
   to start Chrome from **Tools → Browser → Start**.
2. Open the homepage: `kensa-cli browser open <HOMEPAGE_URL>` (replace with the
   project's URL — ask the user once if you don't know it).
3. Read the hero headline: `kensa-cli browser text "h1" --format json`. Report the
   text and whether it matches what's expected.
4. Enumerate the primary navigation links:
   `kensa-cli browser eval "JSON.stringify([...document.querySelectorAll('nav a')].map(a=>({txt:a.innerText.trim(),href:a.getAttribute('href')})))"`.
   List them for the user.
5. Capture a baseline screenshot:
   `kensa-cli browser screenshot --out .tms/attachments/rt-001-home.png`.
6. Report: headline OK/changed, nav links found, screenshot path. If anything is
   broken (missing `h1`, JS console errors, dead nav), file a defect case with
   `kensa-cli new` per the `kensa-browser` report-back loop.
